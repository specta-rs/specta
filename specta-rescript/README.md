# specta-rescript

Export your Rust types to [ReScript](https://rescript-lang.org), built on top of [Specta](https://github.com/specta-rs/specta).

## Features

- **Idiomatic ReScript types** — records, polymorphic variants, regular variants
- **Built-in `result<t, e>` detection** — any `Ok(T)`/`Err(E)` enum maps to ReScript's native `result`
- **Polymorphic variants** — all-unit enums become `[#A | #B | #C]`
- **Generic type parameters** — Rust `T` becomes ReScript `'t`
- **Auxiliary record types** — named-field enum variants get a generated record type
- **Serde integration** — applies `specta-serde` transformations before export
- **Documentation and deprecations** — type, field, and variant metadata becomes ReScript doc comments
- **Recursive types** — self-recursive and mutually recursive types use `type rec` / `and`

## Quick Start

```toml
[dependencies]
specta = { version = "2.0.0-rc.26", features = ["derive"] }
specta-rescript = "0.0.1"
specta-serde = "0.0.13"
```

Define your types:

```rust
use specta::{Type, Types};
use specta_rescript::ReScript;

#[derive(Type)]
struct User {
    id: u32,
    name: String,
    email: Option<String>,
    role: UserRole,
}

#[derive(Type)]
enum UserRole {
    Guest,
    Moderator,
    Admin,
}

#[derive(Type)]
enum ApiResult<T, E> {
    Ok(T),
    Err(E),
}
```

Generate ReScript:

```rust
fn main() {
    let types = Types::default()
        .register::<User>()
        .register::<UserRole>()
        .register::<ApiResult<String, String>>();

    ReScript::default().export_to("./Types.res", &types).unwrap();
}
```

Output:

```rescript
// This file has been generated by Specta. DO NOT EDIT.

type userRole = [#Guest | #Moderator | #Admin]

type user = {
  id: int,
  name: string,
  email: option<string>,
  role: userRole,
}

type apiResult<'t, 'e> = result<'t, 'e>
```

## Type Mapping

| Rust | ReScript | Notes |
|---|---|---|
| `i8`, `i16`, `i32`, `u8`, `u16` | `int` | 32-bit ReScript integer |
| `i64`, `u32`, `u64`, `isize`, `usize` | `bigint` | Values may exceed ReScript `int` |
| `f32`, `f64` | `float` | |
| `bool` | `bool` | |
| `char`, `String` | `string` | |
| `Option<T>` | `option<t>` | ReScript built-in |
| `Vec<T>`, `[T; N]` | `array<t>` | |
| `HashMap<String, V>` | `dict<v>` | String keys only |
| `(T1, T2)` | `(t1, t2)` | |
| Unit struct | `unit` | |
| Newtype struct `Foo(T)` | `type foo = t` | Unwrapped |
| Named struct | `type foo = { field: t, ... }` | Record |
| All-unit enum | `type foo = [#A \| #B]` | Polymorphic variants |
| Enum with data | `type foo = \| A \| B(t)` | Regular variants |
| `Ok(T)`/`Err(E)` enum | `result<t, e>` | ReScript built-in |
| Generic param `T` | `'t` | Apostrophe-prefixed |

## Limitations

The following are not currently supported and will return an error at export time:

| Unsupported | Reason |
|---|---|
| `i128`, `u128` | No equivalent integer type in ReScript |
| `f16` | No equivalent float type in ReScript |
| `HashMap<K, V>` with non-string keys | ReScript `dict` only supports string keys |
| Opaque references (`#[specta(opaque)]`) | No representation in the exporter |

Current behavioral limitations:

| Not yet implemented | Notes |
|---|---|
| Exporter configuration | No naming convention or formatting options (unlike specta-swift) |
| `#[serde(tag)]`, `#[serde(untagged)]` | Complex enum representations are not mapped |
